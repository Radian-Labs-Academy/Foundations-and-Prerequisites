# Radian Labs Academy — Foundations & Prerequisites

Reference code and solutions for the **Radian Labs AMR program**.
Repo: <https://github.com/Radian-Labs-Academy/Foundations-and-Prerequisites>

Every snippet you copy during a lesson, and every solution to a lab or project, lives here —
organized to mirror the workspace you build on your own machine.

[![Verify Programs](https://github.com/Radian-Labs-Academy/Foundations-and-Prerequisites/actions/workflows/ci.yml/badge.svg)](https://github.com/Radian-Labs-Academy/Foundations-and-Prerequisites/actions/workflows/ci.yml)

> You don't run your coursework from inside this repo. You build your own workspace at
> `~/radian_ws` and copy files into it. This repo is the *source of truth* you copy *from*.

## The Radian Labs Workspace Convention

Every lesson and every future course uses one workspace, on your machine, at `~/radian_ws`:

```
~/radian_ws/
├── practice/          # hand-written drills (Linux, early lessons) — one folder per lesson
│   └── module-01/
│       └── lesson-02/
├── src/               # ROS2 packages (from Lesson 1.10 onward), all named radian_*
│   └── radian_intro/
├── build/  install/  log/    # generated by colcon — never edit by hand
└── README.md
```

## How This Repo Is Organized

| Folder | What's in it | You… |
|--------|--------------|------|
| `reference/` | The code each lesson tells you to copy in (snippets, starter files). | **copy** into `~/radian_ws/practice/...` |
| `solutions/` | Worked answers to every lab, project, and to-do. | **try first**, then compare |
| `src/`       | ROS2 packages (`radian_*`), added from Lesson 1.10 onward. | build with `colcon` |

Both `reference/` and `solutions/` mirror the `moduleXX/lessonYY` layout, so a file's path
tells you exactly where it belongs.

## How To Use This Repo

```bash
# Clone it once, into a short folder name in your home directory:
cd ~
git clone https://github.com/Radian-Labs-Academy/Foundations-and-Prerequisites.git ~/radian-labs-foundations

# Each lesson then tells you which file to copy into your workspace, e.g.:
cp ~/radian-labs-foundations/reference/module-01-linux/lesson-02-terminal/hello_radian.sh \
   ~/radian_ws/practice/module-01/lesson-02/

# Pull the latest reference code any time:
cd ~/radian-labs-foundations && git pull
```

## Continuous Verification (CI/CD)

Every push and pull request runs **[`.github/workflows/ci.yml`](.github/workflows/ci.yml)**, which
compiles AND runs **every program in the repo**, checking expected output:

- **Shell** — `bash -n` + `shellcheck` on every `*.sh`
- **Python** — `python -m py_compile` on every `*.py`
- **C++** — `g++ -std=c++17 -Wall -Wextra` on every standalone `*.cpp`/`*.cc`
- **Execute** — runs every `tests/**/test_*.sh`, asserting each program's expected output
- **Compose** — `docker compose config` on every `docker-compose*.yml`
- **Image** — `docker build` of `radian-amr-dev`
- **ROS2** — `colcon build` of every package under `src/` in a `ros:jazzy` container

Each job discovers files of its kind and skips cleanly when there are none yet, so the checks
grow automatically as the course adds more code. If CI is green, everything in the repo builds
and runs as expected.

## Carry-Forward Artifacts

This repo isn't just teaching snippets — each concept is taught properly, and the hands-on
**"TO DO"** prepares a **reusable artifact** that later courses actually consume, e.g.:

- **`docker/radian-amr-dev/`** — the dev image (ROS2 + tooling). Later courses build `FROM radian-amr-dev`.
- **`src/radian_intro/`** — the first ROS2 package; later packages join it under `src/`.
- **`reference/.../radian_helpers.sh`** — the shell environment every course assumes.

CI rebuilds/retests each artifact on every push, so nothing the program depends on can silently rot.
