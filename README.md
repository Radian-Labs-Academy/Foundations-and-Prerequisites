## How This Repo Is Organized

| Folder | What's in it | You‚Ä¶ |
|--------|--------------|------|
| `reference/` | The code each lesson tells you to copy in (snippets, starter files). | **copy** into `~/radian_ws/practice/...` |
| `solutions/` | Worked answers to every lab, project, and to-do. | **try first**, then compare |

Both `reference/` and `solutions/` mirror the `moduleXX/lessonYY` layout, so a file's path
tells you exactly where it belongs.

## How To Use This Repo

```bash
# Clone it once, into a short folder name in your home directory:
cd ~
git clone https://github.com/Radian-Labs-Academy/Foundations-and-Prerequisites.git ~/radian-labs-foundations

# Each lesson then tells you which file to copy into your workspace, e.g.:
cp ~/radian-labs-foundations/reference/module-01-linux/lesson-02-terminal/hello_radian.sh \
   ~/radian_ws/practice/module-01/lesson-02/

# Pull the latest reference code any time:
cd ~/radian-labs-foundations && git pull
```

## Continuous Verification (CI/CD)

Every push and pull request runs **[`.github/workflows/ci.yml`](.github/workflows/ci.yml)**, which
compiles and verifies **every program in the repo**:

- **Shell** ‚Äî `bash -n` (syntax) + `shellcheck` on every `*.sh`
- **Python** ‚Äî `python -m py_compile` on every `*.py`
- **C++** ‚Äî `g++ -std=c++17 -Wall -Wextra` on every standalone `*.cpp`/`*.cc`
- **ROS2** ‚Äî `colcon build` + `colcon test` on any package under `src/` (auto-enables once packages exist)

Each job discovers files of its kind and skips cleanly when there are none yet, so the checks
grow automatically as the course adds more code. If CI is green, everything in the repo builds.
